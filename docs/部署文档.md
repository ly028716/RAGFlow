# 部署文档

## 概述

本文档说明如何部署AI智能助手系统（包含Phase 2浏览器自动化采集功能）。

## 系统要求

### 硬件要求
- CPU: 4核心以上
- 内存: 8GB以上（推荐16GB）
- 磁盘: 50GB以上可用空间

### 软件要求
- Docker 20.10+
- Docker Compose 2.0+
- Git 2.30+

## 快速开始

### 1. 克隆项目

```bash
git clone <repository-url>
cd RAGAgentLangChain
```

### 2. 配置环境变量

#### 开发环境

```bash
cd backend
cp .env.example .env
```

编辑`.env`文件，配置以下必需项：

```bash
# 必需配置
DASHSCOPE_API_KEY=your-dashscope-api-key-here
SECRET_KEY=your-secret-key-here-change-in-production

# 数据库配置（使用Docker默认值）
MYSQL_ROOT_PASSWORD=rootpassword
MYSQL_DATABASE=ai_assistant
MYSQL_USER=ai_user
MYSQL_PASSWORD=ai_password

# Redis配置
REDIS_PASSWORD=
```

#### 生产环境

```bash
cd backend
cp .env.prod.example .env.prod
```

编辑`.env.prod`文件，配置所有生产环境变量。

### 3. 部署

#### 开发环境部署

```bash
cd backend
chmod +x scripts/deploy-dev.sh
./scripts/deploy-dev.sh
```

#### 生产环境部署

```bash
cd backend
chmod +x scripts/deploy-prod.sh
./scripts/deploy-prod.sh
```

## 详细配置说明

### 环境变量说明

#### 应用配置

| 变量名 | 说明 | 默认值 | 必需 |
|--------|------|--------|------|
| APP_NAME | 应用名称 | AI智能助手系统 | 否 |
| APP_VERSION | 应用版本 | 0.1.0 | 否 |
| DEBUG | 调试模式 | True | 否 |
| ENVIRONMENT | 运行环境 | development | 否 |

#### 数据库配置

| 变量名 | 说明 | 默认值 | 必需 |
|--------|------|--------|------|
| DATABASE_URL | 数据库连接URL | - | 是 |
| DB_POOL_SIZE | 连接池大小 | 10 | 否 |
| DB_MAX_OVERFLOW | 最大溢出连接数 | 20 | 否 |

#### LLM配置

| 变量名 | 说明 | 默认值 | 必需 |
|--------|------|--------|------|
| DASHSCOPE_API_KEY | 通义千问API密钥 | - | 是 |
| TONGYI_MODEL_NAME | 模型名称 | qwen-turbo | 否 |
| TONGYI_TEMPERATURE | 温度参数 | 0.7 | 否 |

#### Web Scraper配置（Phase 2新增）

| 变量名 | 说明 | 默认值 | 必需 |
|--------|------|--------|------|
| SCRAPER_MAX_CONCURRENT_TASKS | 最大并发任务数 | 3 | 否 |
| SCRAPER_DEFAULT_TIMEOUT | 默认超时时间(ms) | 30000 | 否 |
| SCRAPER_URL_WHITELIST | URL白名单(逗号分隔) | - | 否 |
| SCRAPER_ENABLE_SCREENSHOT | 启用截图功能 | False | 否 |

### Docker配置说明

#### 开发环境 (docker-compose.yml)

- **MySQL**: 端口3306，数据持久化
- **Redis**: 端口6379，AOF持久化
- **Backend**: 端口8000，热重载
- **Frontend**: 端口5173，开发服务器

#### 生产环境 (docker-compose.prod.yml)

- **MySQL**: 仅内网访问，数据持久化
- **Redis**: 仅内网访问，密码保护
- **Backend**: 仅内网访问
- **Frontend**: 端口80，Nginx反向代理

## 服务管理

### 查看服务状态

```bash
# 开发环境
docker-compose ps

# 生产环境
docker-compose -f docker-compose.prod.yml ps
```

### 查看日志

```bash
# 查看所有服务日志
docker-compose logs -f

# 查看特定服务日志
docker-compose logs -f backend
docker-compose logs -f frontend

# 生产环境
docker-compose -f docker-compose.prod.yml logs -f
```

### 重启服务

```bash
# 重启所有服务
docker-compose restart

# 重启特定服务
docker-compose restart backend

# 生产环境
docker-compose -f docker-compose.prod.yml restart
```

### 停止服务

```bash
# 停止所有服务
docker-compose down

# 停止并删除数据卷（危险操作！）
docker-compose down -v

# 生产环境
docker-compose -f docker-compose.prod.yml down
```

## 数据库管理

### 运行迁移

```bash
# 开发环境
docker-compose exec backend alembic upgrade head

# 生产环境
docker-compose -f docker-compose.prod.yml exec backend alembic upgrade head
```

### 创建新迁移

```bash
docker-compose exec backend alembic revision --autogenerate -m "描述"
```

### 回滚迁移

```bash
# 回滚一个版本
docker-compose exec backend alembic downgrade -1

# 回滚到特定版本
docker-compose exec backend alembic downgrade <revision>
```

### 数据库备份

```bash
# 备份数据库
docker-compose exec mysql mysqldump -u${MYSQL_USER} -p${MYSQL_PASSWORD} ${MYSQL_DATABASE} > backup.sql

# 恢复数据库
docker-compose exec -T mysql mysql -u${MYSQL_USER} -p${MYSQL_PASSWORD} ${MYSQL_DATABASE} < backup.sql
```

## 监控和健康检查

### 健康检查端点

- **后端健康检查**: http://localhost:8000/api/v1/system/health
- **前端健康检查**: http://localhost:5173 (开发) / http://localhost (生产)
- **监控指标**: http://localhost:9090/metrics

### 查看容器健康状态

```bash
docker-compose ps
```

健康状态说明：
- `healthy`: 服务正常运行
- `unhealthy`: 服务异常
- `starting`: 服务启动中

## 故障排查

### 常见问题

#### 1. 容器启动失败

```bash
# 查看容器日志
docker-compose logs <service-name>

# 检查容器状态
docker-compose ps

# 重新构建镜像
docker-compose build --no-cache <service-name>
```

#### 2. 数据库连接失败

- 检查MySQL容器是否健康
- 验证数据库连接配置
- 检查网络连接

```bash
docker-compose exec backend ping mysql
```

#### 3. Playwright浏览器问题

```bash
# 重新安装浏览器
docker-compose exec backend playwright install chromium --with-deps

# 检查浏览器路径
docker-compose exec backend which chromium
```

#### 4. 前端无法访问后端API

- 检查CORS配置
- 验证API基础URL配置
- 检查网络连接

### 日志位置

- **后端日志**: `backend/logs/app.log`
- **Docker日志**: `docker-compose logs`
- **Nginx日志**: 容器内 `/var/log/nginx/`

## 性能优化

### 数据库优化

1. 调整连接池大小：
```bash
DB_POOL_SIZE=20
DB_MAX_OVERFLOW=40
```

2. 启用查询缓存
3. 定期清理日志表

### Redis优化

1. 配置最大内存：
```yaml
command: redis-server --maxmemory 2gb --maxmemory-policy allkeys-lru
```

2. 启用持久化：
```yaml
command: redis-server --appendonly yes
```

### Web Scraper优化

1. 调整并发任务数：
```bash
SCRAPER_MAX_CONCURRENT_TASKS=5
```

2. 禁用截图功能（生产环境）：
```bash
SCRAPER_ENABLE_SCREENSHOT=False
```
