## 2. 用户故事

### 2.1 用户故事 #1: 查看 OpenClaw 连接状态

**作为** RAG Agent 系统的用户
**我想要** 在界面上实时看到 OpenClaw 的连接状态
**以便于** 了解系统当前的工作模式和可用功能

#### 验收标准

**功能验收**:
- [ ] Given 用户打开系统 When OpenClaw 正常连接 Then 顶部导航栏显示绿色状态指示器，文本为"OpenClaw 已连接"
- [ ] Given 用户打开系统 When OpenClaw 连接中 Then 显示黄色状态指示器，文本为"OpenClaw 连接中..."
- [ ] Given 用户打开系统 When OpenClaw 不可用 Then 显示红色状态指示器，文本为"OpenClaw 不可用"
- [ ] Given 状态指示器显示 When 鼠标悬停 Then 显示 Tooltip，包含版本号和最后检查时间
- [ ] Given 状态指示器显示 When 点击状态指示器 Then 打开详情弹窗，显示完整的连接信息

**UI 验收**:
- [ ] 状态指示器位于顶部导航栏右侧，不遮挡主要功能
- [ ] 绿色状态使用颜色 `#52c41a`
- [ ] 黄色状态使用颜色 `#faad14`
- [ ] 红色状态使用颜色 `#ff4d4f`
- [ ] 状态点直径 8px，文本使用 14px 字体
- [ ] 移动端状态指示器可折叠到菜单中

**性能验收**:
- [ ] 状态指示器渲染时间 < 100ms
- [ ] 状态更新动画流畅，无卡顿
- [ ] 详情弹窗打开时间 < 200ms

#### 边界情况

- 首次加载时状态为"未知"，显示灰色
- 网络断开时状态变为"不可用"
- 浏览器标签页切换回来时自动刷新状态

### 2.2 用户故事 #2: 查看 OpenClaw 详细信息

**作为** RAG Agent 系统的用户
**我想要** 查看 OpenClaw 的详细连接信息
**以便于** 了解版本、运行时间、Gateway 地址等技术细节

#### 验收标准

**功能验收**:
- [ ] Given 用户点击状态指示器 When 弹窗打开 Then 显示以下信息：状态、版本、运行时间、Gateway URL、最后检查时间
- [ ] Given 详情弹窗打开 When OpenClaw 状态为"不可用" Then 显示错误信息和建议操作
- [ ] Given 详情弹窗打开 When 点击"重新连接"按钮 Then 立即触发健康检查，显示加载状态
- [ ] Given 重新连接成功 When 健康检查完成 Then 显示成功提示，更新状态信息
- [ ] Given 重新连接失败 When 健康检查失败 Then 显示失败提示，保持原状态

**UI 验收**:
- [ ] 弹窗宽度 400px，居中显示
- [ ] 信息以键值对形式展示，左对齐
- [ ] 运行时间格式化为"X小时X分钟"
- [ ] 最后检查时间显示为相对时间（如"5秒前"）
- [ ] "重新连接"按钮在加载时显示 loading 状态

**交互验收**:
- [ ] 点击弹窗外部区域关闭弹窗
- [ ] 按 ESC 键关闭弹窗
- [ ] 重新连接时按钮禁用，防止重复点击

#### 边界情况

- Gateway URL 过长时显示省略号
- 运行时间超过 24 小时显示为"X天X小时"
- 版本号为空时显示"未知"

### 2.3 用户故事 #3: 接收状态变化通知

**作为** RAG Agent 系统的用户
**我想要** 在 OpenClaw 状态变化时收到通知
**以便于** 及时了解系统工作模式的变化

#### 验收标准

**功能验收**:
- [ ] Given OpenClaw 从"不可用"变为"正常" When 状态变化 Then 显示成功通知："OpenClaw 已连接，版本: X.X.X"
- [ ] Given OpenClaw 从"正常"变为"不可用" When 状态变化 Then 显示警告通知："OpenClaw 暂时不可用，已切换到知识库模式，功能不受影响"
- [ ] Given OpenClaw 从"不可用"恢复为"正常" When 状态变化 Then 显示成功通知："OpenClaw 已恢复，现在可以使用完整功能"
- [ ] Given 用户关闭通知 When 点击关闭按钮 Then 通知立即消失
- [ ] Given 通知显示 When 超过设定时长 Then 通知自动消失

**UI 验收**:
- [ ] 成功通知使用绿色主题
- [ ] 警告通知使用黄色主题
- [ ] 通知显示在右上角
- [ ] 通知包含标题和描述
- [ ] 通知有关闭按钮

**配置验收**:
- [ ] 用户可以在设置中关闭状态变化通知
- [ ] 用户可以配置通知显示时长（默认 3 秒）
- [ ] 配置保存在 localStorage，刷新后保持

**性能验收**:
- [ ] 通知显示延迟 < 100ms
- [ ] 通知动画流畅，无卡顿
- [ ] 同时最多显示 3 个通知

#### 边界情况

- 短时间内多次状态变化只显示最后一次通知
- 用户关闭通知功能后不再显示任何通知
- 浏览器标签页在后台时不显示通知

### 2.4 用户故事 #4: 使用 OpenClaw 发送消息

**作为** RAG Agent 系统的用户
**我想要** 在 OpenClaw 正常工作时发送消息并获得响应
**以便于** 使用完整的 AI 功能（包括浏览器采集等）

#### 验收标准

**功能验收**:
- [ ] Given OpenClaw 状态为"正常" When 用户发送消息 Then 消息旁显示 🔵 OpenClaw 模式标识
- [ ] Given 消息发送成功 When OpenClaw 返回响应 Then 显示 AI 回复，包含执行时间
- [ ] Given 消息处理中 When 等待响应 Then 显示加载状态："OpenClaw 处理中..."
- [ ] Given 消息发送失败 When OpenClaw 返回错误 Then 显示错误提示，自动降级到知识库模式

**UI 验收**:
- [ ] OpenClaw 模式标识使用蓝色圆点图标 🔵
- [ ] 消息卡片显示模式标识和执行时间
- [ ] 加载状态显示动画效果
- [ ] 错误提示使用红色文本

**性能验收**:
- [ ] P50 响应时间 < 1s
- [ ] P95 响应时间 < 3s
- [ ] P99 响应时间 < 5s
- [ ] 超过 30s 自动超时并降级

#### 边界情况

- 消息内容为空时禁用发送按钮
- 消息长度超过 10000 字符时显示警告
- 网络断开时立即显示错误提示

### 2.5 用户故事 #5: OpenClaw 不可用时自动降级

**作为** RAG Agent 系统的用户
**我想要** 在 OpenClaw 不可用时系统自动切换到知识库模式
**以便于** 继续使用系统，不影响基本的问答功能

#### 验收标准

**功能验收**:
- [ ] Given OpenClaw 状态为"不可用" When 用户发送消息 Then 自动使用知识库模式处理
- [ ] Given 使用知识库模式 When 消息返回 Then 消息旁显示 📚 知识库模式标识
- [ ] Given 降级到知识库模式 When 首次发送消息 Then 显示降级提示条："当前使用知识库模式，OpenClaw 暂时不可用，部分功能受限"
- [ ] Given 降级提示条显示 When 用户点击关闭 Then 提示条消失，本次会话不再显示
- [ ] Given 降级模式 When 输入框上方 Then 显示功能受限说明

**UI 验收**:
- [ ] 知识库模式标识使用书本图标 📚
- [ ] 降级提示条使用黄色警告样式
- [ ] 功能受限说明列出不可用功能：实时网页采集、浏览器自动化、混合推理
- [ ] 功能受限说明列出可用功能：知识库问答

**用户体验验收**:
- [ ] 降级过程无感知，不阻断用户操作
- [ ] 降级提示清晰，不引起用户困惑
- [ ] 知识库模式响应速度不受影响

**性能验收**:
- [ ] 降级判断时间 < 100ms
- [ ] 知识库模式响应时间与正常模式一致

#### 边界情况

- 纯知识库查询不显示降级提示（无感知降级）
- 降级状态持久化，刷新页面后保持
- 降级期间每 5 分钟自动尝试恢复连接

### 2.6 用户故事 #6: OpenClaw 恢复后自动切换

**作为** RAG Agent 系统的用户
**我想要** 在 OpenClaw 恢复后系统自动切换回完整功能模式
**以便于** 使用所有高级功能，无需手动操作

#### 验收标准

**功能验收**:
- [ ] Given 系统处于降级状态 When OpenClaw 恢复正常 Then 状态指示器变为绿色
- [ ] Given OpenClaw 恢复 When 状态变化 Then 显示恢复通知："OpenClaw 已恢复，现在可以使用完整功能"
- [ ] Given OpenClaw 恢复 When 用户发送新消息 Then 自动使用 OpenClaw 模式处理
- [ ] Given 降级提示条显示 When OpenClaw 恢复 Then 提示条自动消失
- [ ] Given 功能受限说明显示 When OpenClaw 恢复 Then 说明自动消失

**UI 验收**:
- [ ] 状态指示器恢复动画流畅（1 秒脉冲动画）
- [ ] 恢复通知使用绿色成功样式
- [ ] 提示条消失使用淡出动画

**自动恢复验收**:
- [ ] 降级后每 5 分钟自动检查一次
- [ ] 恢复检查不影响用户操作
- [ ] 恢复检查失败不显示错误提示

**性能验收**:
- [ ] 恢复检查时间 < 1s
- [ ] 状态切换延迟 < 200ms
- [ ] 恢复通知显示延迟 < 100ms

#### 边界情况

- 恢复检查期间用户手动重连成功，停止自动检查
- 恢复后立即发送的消息使用 OpenClaw 模式
- 恢复通知显示 3 秒后自动消失

### 2.7 用户故事 #7: 手动重新连接 OpenClaw

**作为** RAG Agent 系统的用户
**我想要** 手动触发 OpenClaw 重新连接
**以便于** 在知道问题已解决后立即恢复功能，无需等待自动恢复

#### 验收标准

**功能验收**:
- [ ] Given 用户打开详情弹窗 When OpenClaw 不可用 Then 显示"重新连接"按钮
- [ ] Given 用户点击"重新连接" When 触发连接 Then 按钮显示 loading 状态，文本变为"连接中..."
- [ ] Given 重新连接成功 When 健康检查通过 Then 显示成功提示："重新连接成功"，关闭弹窗
- [ ] Given 重新连接失败 When 健康检查失败 Then 显示失败提示："重新连接失败，请稍后再试"，保持弹窗打开
- [ ] Given 重新连接中 When 用户再次点击按钮 Then 按钮禁用，不触发重复请求

**UI 验收**:
- [ ] "重新连接"按钮使用主题色
- [ ] loading 状态显示旋转图标
- [ ] 成功提示使用绿色样式
- [ ] 失败提示使用红色样式

**交互验收**:
- [ ] 重新连接期间不能关闭弹窗
- [ ] 重新连接完成后可以关闭弹窗
- [ ] 连续失败 3 次后按钮禁用 1 分钟

**性能验收**:
- [ ] 重新连接响应时间 < 3s
- [ ] 按钮状态切换延迟 < 50ms

#### 边界情况

- 重新连接期间网络断开，显示网络错误提示
- 重新连接超时（3 秒），显示超时提示
- 重新连接成功后重置重试计数器

### 2.8 用户故事 #8: 配置状态通知选项

**作为** RAG Agent 系统的用户
**我想要** 配置状态变化通知的行为
**以便于** 根据个人偏好调整通知方式

#### 验收标准

**功能验收**:
- [ ] Given 用户打开设置页面 When 进入"通知设置" Then 显示 OpenClaw 通知配置选项
- [ ] Given 通知配置显示 When 用户切换"显示状态变化通知" Then 立即生效，保存到 localStorage
- [ ] Given 通知配置显示 When 用户调整"通知显示时长" Then 立即生效，下次通知使用新时长
- [ ] Given 通知配置显示 When 用户切换"自动重试连接" Then 立即生效，影响降级后的重试行为
- [ ] Given 通知配置显示 When 用户调整"健康检查间隔" Then 立即生效，下次检查使用新间隔

- [ ] "通知显示时长"：滑块，范围 1-10 秒，默认 3 秒
- [ ] "自动重试连接"：开关，默认开启
- [ ] "健康检查间隔"：滑块，范围 10-120 秒，默认 30 秒

**UI 验收**:
- [ ] 配置选项使用 Element Plus 组件
- [ ] 每个选项有清晰的标签和说明
- [ ] 滑块显示当前值
- [ ] 配置变更有保存提示

**持久化验收**:
- [ ] 配置保存到 localStorage
- [ ] 刷新页面后配置保持
- [ ] 清除浏览器数据后恢复默认值

#### 边界情况

- 健康检查间隔设置过短（< 10 秒）显示警告
- 通知显示时长设置为 0 时等同于关闭通知
- 配置保存失败时显示错误提示
