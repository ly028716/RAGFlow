## 编写新测试

### 测试文件命名规范

- 测试文件名：`test_<module_name>.py`
- 测试类名：`Test<ClassName>`
- 测试方法名：`test_<function_name>_<scenario>`

### 测试结构模板

```python
"""
模块功能测试

测试范围：
- 功能点1
- 功能点2
- 功能点3
"""
import pytest
from unittest.mock import Mock, AsyncMock, patch

from app.module import TargetClass


@pytest.fixture
def sample_data():
    """测试数据fixture"""
    return {
        "field1": "value1",
        "field2": "value2"
    }


class TestTargetClass:
    """目标类测试"""

    def test_function_success(self, sample_data):
        """测试成功场景"""
        # Arrange - 准备测试数据
        instance = TargetClass()

        # Act - 执行测试
        result = instance.function(sample_data)

        # Assert - 验证结果
        assert result is not None
        assert result.field1 == "value1"

    def test_function_error(self):
        """测试错误场景"""
        instance = TargetClass()

        with pytest.raises(ValueError, match="错误信息"):
            instance.function(invalid_data)

    @pytest.mark.asyncio
    async def test_async_function(self):
        """测试异步函数"""
        instance = TargetClass()

        result = await instance.async_function()

        assert result is not None
```

### Mock使用指南

#### Mock同步函数

```python
from unittest.mock import Mock, patch

# Mock对象
mock_obj = Mock()
mock_obj.method.return_value = "result"

# Mock函数
with patch('app.module.function') as mock_func:
    mock_func.return_value = "result"
    # 测试代码
```

#### Mock异步函数

```python
from unittest.mock import AsyncMock, patch

# Mock异步对象
mock_obj = AsyncMock()
mock_obj.async_method.return_value = "result"

# Mock异步函数
with patch('app.module.async_function') as mock_func:
    mock_func.return_value = AsyncMock(return_value="result")
    # 测试代码
```

#### Mock数据库

```python
@pytest.fixture
def mock_db():
    """Mock数据库会话"""
    db = Mock()
    db.query.return_value.filter.return_value.first.return_value = mock_object
    return db
```

#### Mock Redis

```python
@pytest.fixture
async def mock_redis():
    """Mock Redis客户端"""
    redis = AsyncMock()
    redis.set.return_value = True
    redis.get.return_value = None
    return redis
```

## 测试最佳实践

### 1. 测试独立性

- 每个测试应该独立运行，不依赖其他测试
- 使用 `beforeEach` 和 `afterEach` 清理测试环境
- 避免共享可变状态

```python
@pytest.fixture(autouse=True)
def setup_and_teardown(db):
    """自动执行的setup和teardown"""
    # Setup
    yield
    # Teardown
    db.rollback()
```

### 2. 测试命名

- 使用描述性的测试名称
- 遵循 "test_<function>_<scenario>" 的命名模式
- 中文测试名称更易读

```python
def test_create_task_success():
    """测试成功创建任务"""
    pass

def test_create_task_invalid_url():
    """测试创建任务时URL无效"""
    pass
```

### 3. AAA模式

遵循 Arrange-Act-Assert 模式：

```python
def test_example():
    # Arrange - 准备测试数据和环境
    user = create_test_user()
    task_data = {"name": "测试任务"}

    # Act - 执行被测试的操作
    result = service.create_task(user.id, task_data)

    # Assert - 验证结果
    assert result.id is not None
    assert result.name == "测试任务"
```

### 4. 异常测试

使用 `pytest.raises` 测试异常：

```python
def test_invalid_input():
    """测试无效输入"""
    with pytest.raises(ValidationError, match="错误信息"):
        service.process_invalid_data()
```

### 5. 参数化测试

使用 `pytest.mark.parametrize` 减少重复代码：

```python
@pytest.mark.parametrize("url,expected", [
    ("https://example.com", True),
    ("http://example.com", True),
    ("ftp://example.com", False),
    ("javascript:alert()", False),
])
def test_url_validation(url, expected):
    """测试URL验证"""
    result = validate_url(url)
    assert result == expected
```

### 6. Fixture复用

使用fixture提供可复用的测试数据：

```python
@pytest.fixture
def test_user(db):
    """创建测试用户"""
    user = User(username="testuser", email="test@example.com")
    db.add(user)
    db.commit()
    db.refresh(user)
    return user

@pytest.fixture
def test_kb(db, test_user):
    """创建测试知识库"""
    kb = KnowledgeBase(name="测试知识库", user_id=test_user.id)
    db.add(kb)
    db.commit()
    db.refresh(kb)
    return kb
```

## 故障排查

### 常见问题

#### 1. 测试数据库连接失败

**问题**: `sqlalchemy.exc.OperationalError: (pymysql.err.OperationalError)`

**解决方案**:
```bash
# 检查MySQL是否运行
docker-compose ps mysql

# 重启MySQL
docker-compose restart mysql

# 检查数据库连接配置
cat .env | grep DATABASE_URL
```

#### 2. Redis连接失败

**问题**: `redis.exceptions.ConnectionError`

**解决方案**:
```bash
# 检查Redis是否运行
docker-compose ps redis

# 重启Redis
docker-compose restart redis
```

#### 3. Playwright浏览器未安装

**问题**: `playwright._impl._api_types.Error: Executable doesn't exist`

**解决方案**:
```bash
# 安装Playwright浏览器
playwright install chromium --with-deps

# 在Docker中
docker-compose exec backend playwright install chromium --with-deps
```

#### 4. 异步测试失败

**问题**: `RuntimeError: Event loop is closed`

**解决方案**:
- 确保使用 `@pytest.mark.asyncio` 装饰器
- 检查 `pytest-asyncio` 是否已安装
- 使用 `AsyncMock` 而不是 `Mock` 处理异步对象

#### 5. Mock未生效

**问题**: Mock的返回值没有被使用

**解决方案**:
- 检查Mock的路径是否正确（应该是使用的位置，不是定义的位置）
- 确保Mock在函数调用之前设置
- 使用 `assert_called_once()` 验证Mock是否被调用
