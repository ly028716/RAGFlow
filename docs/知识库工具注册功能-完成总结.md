### 1. 数据库设计

**表结构**:
- `openclaw_tools` - 工具配置表
- `openclaw_tool_calls` - 工具调用记录表

**迁移文件**: `backend/migrations/versions/009_add_openclaw_tools_tables.py`

**字段说明**:
```sql
openclaw_tools:
  - id: 工具ID
  - name: 工具名称（唯一）
  - display_name: 显示名称
  - description: 工具描述
  - endpoint_url: 端点URL
  - method: HTTP方法（GET/POST）
  - auth_type: 认证类型（api_token）
  - auth_config: 认证配置（JSON）
  - parameters_schema: 参数Schema（JSON Schema）
  - response_schema: 响应Schema（JSON Schema）
  - status: 工具状态（active/inactive/deleted）
  - is_builtin: 是否为内置工具
  - created_by: 创建者用户ID
  - created_at: 创建时间
  - updated_at: 更新时间

openclaw_tool_calls:
  - id: 调用记录ID
  - tool_id: 工具ID（外键）
  - agent_id: Agent ID
  - user_id: 用户ID
  - request_params: 请求参数（JSON）
  - response_data: 响应数据（JSON）
  - status: 调用状态（success/failed/timeout）
  - error_message: 错误信息
  - execution_time: 执行时间（秒）
  - created_at: 调用时间
```

### 3. API端点

#### 工具管理端点（需要管理员权限）

**POST /api/v1/openclaw/tools/register**
- 功能: 注册新工具
- 权限: 管理员
- 请求体:
```json
{
  "name": "query_knowledge_base",
  "display_name": "知识库查询",
  "description": "查询RAG知识库中的相关文档",
  "endpoint_url": "http://localhost:8000/api/v1/tools/query-kb",
  "method": "POST",
  "auth_type": "api_token",
  "auth_config": {
    "header_name": "X-API-Token"
  },
  "parameters_schema": { ... },
  "response_schema": { ... }
}
```

**GET /api/v1/openclaw/tools**
- 功能: 获取工具列表
- 权限: 登录用户
- 查询参数:
  - `status_filter`: 状态过滤（active/inactive/deleted）
  - `is_builtin`: 是否为内置工具
  - `skip`: 跳过数量
  - `limit`: 返回数量

**GET /api/v1/openclaw/tools/{tool_id}**
- 功能: 获取工具详情
- 权限: 登录用户

#### 知识库查询工具端点（供OpenClaw调用）

**POST /api/v1/tools/query-kb**
- 功能: 查询知识库（供OpenClaw Agent调用）
- 认证: API Token（请求头 `X-API-Token`）
- 请求体:
```json
{
  "query": "什么是Python？",
  "knowledge_base_ids": [1, 2],
  "top_k": 5,
  "similarity_threshold": 0.7
}
```
- 响应:
```json
{
  "success": true,
  "query": "什么是Python？",
  "results": [
    {
      "content": "Python是一种高级编程语言...",
      "similarity_score": 0.92,
      "document_id": 1,
      "document_name": "Python教程.pdf",
      "knowledge_base_id": 1,
      "knowledge_base_name": "技术文档"
    }
  ],
  "total_results": 5
}
```

### 4. 配置管理

**配置类**: `OpenClawSettings` (backend/app/config.py)

**环境变量**:
```bash
# OpenClaw Gateway配置
OPENCLAW_GATEWAY_URL=http://localhost:19001
OPENCLAW_TIMEOUT=30.0
OPENCLAW_MAX_RETRIES=3
OPENCLAW_ENABLED=true

# API Token配置（逗号分隔多个token）
OPENCLAW_API_TOKENS=token1,token2,token3
```

**配置说明**:
- `gateway_url`: OpenClaw Gateway地址
- `timeout`: 请求超时时间（秒）
- `max_retries`: 最大重试次数
- `enabled`: 是否启用OpenClaw集成
- `api_tokens`: API Token列表，用于OpenClaw调用时的身份验证

## 📖 使用指南

### OpenClaw Agent调用流程

```
┌─────────────┐
│   飞书用户   │
└──────┬──────┘
       │ 1. 发送消息
       ↓
┌─────────────────────┐
│  OpenClaw Gateway   │
│  (WSL:18789)        │
└──────┬──────────────┘
       │ 2. Agent处理
       │ 3. 调用工具
       ↓
┌─────────────────────────────┐
│  AI智能助手 (FastAPI:8000)  │
│  POST /api/v1/tools/query-kb│
│  Header: X-API-Token         │
└──────┬──────────────────────┘
       │ 4. RAG检索
       ↓
┌─────────────────┐
│  知识库 (Chroma) │
└──────┬──────────┘
       │ 5. 返回结果
       ↓
┌─────────────┐
│   飞书用户   │
└─────────────┘
```

### 工具注册最佳实践

1. **工具命名规范**
   - 使用小写字母和下划线
   - 名称应具有描述性
   - 示例: `query_knowledge_base`, `search_documents`

2. **端点URL配置**
   - 开发环境: `http://localhost:8000/api/v1/tools/...`
   - 生产环境: `https://api.example.com/api/v1/tools/...`
   - 确保OpenClaw Gateway可以访问该URL

3. **认证配置**
   - 使用API Token认证（推荐）
   - Token应足够长且随机（建议64字符）
   - 定期轮换Token
   - 不同环境使用不同Token

4. **Schema设计**
   - 使用JSON Schema标准
   - 提供清晰的字段描述
   - 设置合理的默认值和约束
   - 示例参考种子脚本中的Schema

### API Token管理

**生成Token**:
```bash
# 方式1: 使用openssl
openssl rand -hex 32

# 方式2: 使用Python
python -c "import secrets; print('sk-' + secrets.token_hex(32))"
```

**配置多个Token**:
```bash
# 用逗号分隔
OPENCLAW_API_TOKENS=sk-token1,sk-token2,sk-token3
```

**Token验证流程**:
1. OpenClaw Agent调用工具时，在请求头中携带Token
2. FastAPI验证Token是否在配置的Token列表中
3. 验证通过则执行工具逻辑，否则返回401错误

## 🔍 故障排查

### 问题1: 工具注册失败

**症状**: POST /api/v1/openclaw/tools/register 返回400错误

**可能原因**:
- 工具名称已存在
- 端点URL格式不正确
- Schema格式不符合JSON Schema标准

**解决方案**:
```bash
# 检查工具是否已存在
curl -X GET "http://localhost:8000/api/v1/openclaw/tools" \
  -H "Authorization: Bearer <token>"

# 验证Schema格式
python -c "import json; json.loads(open('schema.json').read())"
```

### 问题2: 知识库查询返回401

**症状**: POST /api/v1/tools/query-kb 返回401 Unauthorized

**可能原因**:
- 未提供API Token
- API Token无效
- Token未配置在环境变量中

**解决方案**:
```bash
# 检查环境变量
echo $OPENCLAW_API_TOKENS

# 验证Token
curl -X POST "http://localhost:8000/api/v1/tools/query-kb" \
  -H "X-API-Token: your-token-here" \
  -H "Content-Type: application/json" \
  -d '{"query": "test"}'
```

### 问题3: 知识库查询返回500

**症状**: POST /api/v1/tools/query-kb 返回500 Internal Server Error

**可能原因**:
- 知识库不存在
- 向量数据库连接失败
- RAG服务异常

**解决方案**:
```bash
# 检查日志
tail -f backend/logs/app.log

# 验证知识库
curl -X GET "http://localhost:8000/api/v1/knowledge-bases" \
  -H "Authorization: Bearer <token>"

# 检查向量数据库
ls -la backend/vector_db/
```
