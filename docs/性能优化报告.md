## 识别的性能问题

### 1. 数据库查询优化 ⚠️ 高优先级

#### 问题1.1: N+1查询问题
**位置**: `backend/app/repositories/web_scraper_log_repository.py:151-164`

**问题描述**:
```python
def get_statistics_by_task(self, task_id: int) -> dict:
    total = self.count_by_task(task_id)
    success = self.count_by_task(task_id, status=LogStatus.SUCCESS)
    failed = self.count_by_task(task_id, status=LogStatus.FAILED)
    running = self.count_by_task(task_id, status=LogStatus.RUNNING)
```

执行4次独立的COUNT查询，效率低下。

**优化方案**:
使用单个SQL查询，通过CASE WHEN进行条件聚合：
```python
def get_statistics_by_task(self, task_id: int) -> dict:
    from sqlalchemy import func, case

    result = self.db.query(
        func.count(WebScraperLog.id).label('total'),
        func.sum(case((WebScraperLog.status == LogStatus.SUCCESS, 1), else_=0)).label('success'),
        func.sum(case((WebScraperLog.status == LogStatus.FAILED, 1), else_=0)).label('failed'),
        func.sum(case((WebScraperLog.status == LogStatus.RUNNING, 1), else_=0)).label('running')
    ).filter(WebScraperLog.task_id == task_id).first()

    total = result.total or 0
    success = result.success or 0
    failed = result.failed or 0
    running = result.running or 0

    return {
        "total": total,
        "success": success,
        "failed": failed,
        "running": running,
        "success_rate": (success / total * 100) if total > 0 else 0,
    }
```

**性能提升**: 4次查询 → 1次查询，减少75%数据库往返

**问题描述**:

**优化方案**:

2. 减少不必要的HTML字符串转换
3. 使用更快的解析器（lxml已是最快）

**性能提升**: 减少30-50%的HTML解析时间

#### 问题2.2: 低效的HTML清洗
**位置**: `backend/app/core/web_scraper.py:350-371`

**问题描述**:
```python
def _clean_html(self, html: str) -> str:
    soup = BeautifulSoup(html, 'lxml')  # 重新解析

    for script in soup(['script', 'style', 'noscript']):
        script.decompose()

    for element in soup.find_all():  # 遍历所有元素
        if not element.get_text(strip=True) and not element.find('img'):
            element.decompose()
```

遍历所有元素效率低。

**优化方案**:
```python
def _clean_html(self, soup: BeautifulSoup) -> BeautifulSoup:
    """直接操作BeautifulSoup对象，避免重新解析"""
    # 移除脚本和样式
    for tag in soup(['script', 'style', 'noscript']):
        tag.decompose()

    # 使用更高效的选择器
    for tag in soup.find_all(lambda t: not t.get_text(strip=True) and not t.find('img')):
        tag.decompose()

    return soup
```

### 3. 缓存策略 ⚠️ 高优先级

#### 问题3.1: 缺少Redis缓存
**位置**: 所有Repository和Service层

**问题描述**:
- 任务列表频繁查询，无缓存
- 任务详情频繁查询，无缓存
- 日志统计频繁计算，无缓存

**优化方案**:
添加Redis缓存层：

```python
# 缓存装饰器
def cache_result(key_prefix: str, ttl: int = 300):
    def decorator(func):
        async def wrapper(*args, **kwargs):
            cache_key = f"{key_prefix}:{args}:{kwargs}"

            # 尝试从缓存获取
            cached = await redis_client.get(cache_key)
            if cached:
                return json.loads(cached)

            # 执行函数
            result = await func(*args, **kwargs)

            # 存入缓存
            await redis_client.setex(cache_key, ttl, json.dumps(result))

            return result
        return wrapper
    return decorator

# 使用示例
@cache_result("task_list", ttl=60)
def list_tasks(self, user_id: int, ...):
    return self.task_repo.get_all_with_relations(...)

@cache_result("task_stats", ttl=300)
def get_log_statistics(self, task_id: int, user_id: int):
    return self.log_repo.get_statistics_by_task(task_id)
```

**缓存失效策略**:
- 任务创建/更新/删除时，清除相关缓存
- 日志创建/更新时，清除统计缓存
- 使用Redis的EXPIRE自动过期

**性能提升**: 减少80-90%的数据库查询

### 4. 查询优化 ⚠️ 中优先级

#### 问题4.1: 缺少预加载
**位置**: `backend/app/repositories/web_scraper_task_repository.py:30-32`

**问题描述**:
```python
def get_by_id(self, task_id: int) -> Optional[WebScraperTask]:
    return self.db.query(WebScraperTask).filter(WebScraperTask.id == task_id).first()
```

没有预加载关联数据，可能导致N+1问题。

**优化方案**:
```python
def get_by_id(self, task_id: int, with_relations: bool = False) -> Optional[WebScraperTask]:
    query = self.db.query(WebScraperTask)

    if with_relations:
        query = query.options(
            joinedload(WebScraperTask.knowledge_base),
            joinedload(WebScraperTask.creator)
        )

    return query.filter(WebScraperTask.id == task_id).first()
```

#### 问题4.2: 分页查询优化
**位置**: 所有Repository的列表查询

**问题描述**:
使用OFFSET分页，大偏移量时性能差。

**优化方案**:
使用游标分页（Cursor-based Pagination）：
```python
def get_all_cursor(
    self,
    cursor: Optional[int] = None,
    limit: int = 100,
    ...
) -> List[WebScraperTask]:
    query = self.db.query(WebScraperTask)

    if cursor:
        query = query.filter(WebScraperTask.id < cursor)

    # 应用其他过滤条件...

    return query.order_by(WebScraperTask.id.desc()).limit(limit).all()
```

### 5. 浏览器资源管理 ⚠️ 低优先级

#### 问题5.1: 浏览器实例管理
**位置**: `backend/app/core/web_scraper.py:139-162`

**问题描述**:
每次采集都创建新的浏览器实例，启动开销大。

**优化方案**:
实现浏览器池：
```python
class BrowserPool:
    def __init__(self, max_size: int = 3):
        self.max_size = max_size
        self.browsers: List[Browser] = []
        self.available: asyncio.Queue = asyncio.Queue()

    async def get_browser(self) -> Browser:
        if self.available.qsize() > 0:
            return await self.available.get()

        if len(self.browsers) < self.max_size:
            browser = await self._create_browser()
            self.browsers.append(browser)
            return browser

        return await self.available.get()

    async def release_browser(self, browser: Browser):
        await self.available.put(browser)
```

**性能提升**: 减少50-70%的浏览器启动时间
